Plivo AI/ML Assignment – LLM Track

Objective

Improve the provided GPT baseline under a fixed parameter budget while keeping the evaluation interface unchanged.

Approach

The work followed an iterative experimental methodology. Rather than modifying multiple components simultaneously, individual architectural and optimization changes were evaluated independently whenever possible.

Experiments Performed

Run Main Change Outcome
0 Baseline Reference
1 AdamW + Weight Decay + Gradient Clipping Improved BPB
2 Warmup + Cosine Scheduler Performance Degraded (Rejected)
3 Weight Tying + Increased Model Capacity Best Performing Configuration
4 Initialization / Final Optimization Final Submitted Model

Final Model

  • AdamW Optimizer
  • Weight Decay = 0.01
  • Gradient Clipping
  • Weight Tying Enabled
  • Scaled Embedding Dimension
  • GPT-style Initialization

Engineering Decisions

A tokenizer redesign was considered because byte-level UTF-8 tokenization is inefficient for multilingual text. However, preserving full compatibility with the supplied evaluation interface for arbitrary UTF-8 input was prioritized. The final submission therefore focuses on optimization and architectural improvements while maintaining complete compatibility with the provided evaluator.
